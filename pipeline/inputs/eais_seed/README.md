# inputs/eais_seed/

Seed-датасет годовых агрегатов российского кинопроката 2019–2025 для
калибровки F1 block bootstrap (Tier E).

## Файлы

- `annual_box_office.csv` — 7 годовых наблюдений со столбцами:
  - `year` — календарный год
  - `total_bo_mln_rub` — валовые кассовые сборы всего проката, млн ₽
  - `viewers_mln` — зрители, млн человек
  - `avg_ticket_rub` — средняя цена билета, ₽
  - `russian_share_pct` — доля российских фильмов в сборах, %
  - `russian_bo_mln_rub` — сборы российских фильмов, млн ₽
  - `n_releases` — количество релизов за год
  - `source_id` — ссылка на `inputs/eais_sources.yaml::sources.reference.*.source_id`
  - `note` — контекст события года

## Источники

Данные сведены из публичных отраслевых отчётов:

1. **Kinometro.ru** — годовые итоги российского кинопроката (2019–2023)
2. **ProfiCinema.com** — аналитические обзоры индустрии (2024–2025)
3. **ТАСС / РБК** — контекст ключевых событий (пандемия, уход Голливуда)

Все цифры — годовые агрегаты, не покрывают детализацию по фильмам.
Детализация — Tier E+ через регистрацию на `ekinobilet.fond-kino.ru`.

## Инварианты seed-датасета

Регрессионные тесты (`tests/test_19_eais_loader.py`):
- 7 строк, годы 2019–2025 непрерывны
- `russian_bo_mln_rub ≈ total_bo_mln_rub × russian_share_pct / 100` (допуск ±3%)
- `total_bo_mln_rub / viewers_mln ≈ avg_ticket_rub` (допуск ±5%)
- `russian_share_pct` монотонно не убывает после 2021 (структурный сдвиг 2022)
- Наблюдается провал 2020 (пандемия): `total_bo_mln_rub[2020] < 0.5 × total_bo_mln_rub[2019]`

## Назначение в пайплайне

F1 bootstrap (Tier E) будет использовать:
- Ряд `total_bo_mln_rub` → распределение year-over-year изменений
- Ряд `russian_share_pct` → структурный сдвиг как отдельный блок
- `viewers_mln` и `avg_ticket_rub` → декомпозиция шока

Семплирование: блочный bootstrap с длиной блока 2 (сохраняет двухлетние
кризисные и восстановительные паттерны).

## Ограничения

1. **Агрегация годами, не кварталами.** Сезонность (новогодние каникулы,
   майские, летние) теряется. Для pipeline это приемлемо — anchor считается
   по годам.
2. **Нет детализации по жанрам.** F4 Gaussian copula будет использовать
   экспертные корреляции, а не эмпирические из данных.
3. **Только 7 точек.** Мал для статистически значимых хвостов; block
   bootstrap компенсирует размер блоками длины 2.
4. **Структурный сдвиг 2022.** Уход Голливуда разорвал ряд 2019–2021 ≠ 2022–2025.
   Bootstrap должен учитывать это (либо раздельные кластеры, либо индикатор
   `post_hollywood_exit`).
