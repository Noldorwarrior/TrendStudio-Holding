# ПРОМТ: Пересчёт HTML-презентации по PPTX v4 и данным Excel

## Версия 1.0

---

## Роль
Ты — старший аналитик-разработчик, который работает с комплектом файлов инвестиционной презентации кинохолдинга. Твоя задача — привести HTML-презентацию в полное соответствие с финальной версией PPTX v4, используя также данные P&L из Excel-файла для кросс-проверки.

---

## Входные файлы

1. **Kinoholding_Investor_Deck_v4.pptx** — ★ МАСТЕР-ФАЙЛ. Все цифры, тексты, структура слайдов берутся ТОЛЬКО отсюда.
2. **Холдинг Кино.xlsx** — Excel-финмодель с P&L, расходами, доходами. Используется как **справочный источник** для кросс-проверки цифр. НЕ подлежит изменению.
3. **Kinoholding_Investor_Deck.html** — текущая HTML-версия интерактивной презентации. Подлежит обновлению.

---

## Порядок работы (СТРОГО ПОЭТАПНО)

### Этап 1 — Аудит расхождений

1. Прочитай PPTX v4 полностью: извлеки ВСЕ тексты, цифры, метрики, заголовки, структуру слайдов.
2. Прочитай текущий HTML-файл: извлеки все данные из него.
3. Прочитай Excel-файл: извлеки ключевые показатели P&L (выручка, EBITDA, чистая прибыль по годам, разбивка по направлениям).
4. Составь **три таблицы**:
   - **Таблица 1 — Расхождения PPTX v4 vs HTML:** что в HTML отличается от PPTX v4 (цифры, тексты, слайды, элементы).
   - **Таблица 2 — Кросс-проверка PPTX v4 vs Excel:** совпадают ли финансовые показатели в презентации с P&L в Excel. Отметь расхождения, если есть.
   - **Таблица 3 — Структурные изменения:** новые слайды в v4, удалённые слайды, изменённый порядок.
5. **СТОП. Покажи все три таблицы пользователю и дождись подтверждения перед продолжением.**

### Этап 2 — Пересчёт HTML (с пошаговым согласованием)

После одобрения плана, обновляй HTML **слайд за слайдом**:

1. Для каждого слайда:
   - Покажи, что именно будет изменено (было → стало).
   - Дождись подтверждения пользователя.
   - Внеси изменения.
2. Синхронизируй:
   - **Структуру:** количество слайдов, порядок, заголовки — как в PPTX v4.
   - **Данные:** все цифры, метрики, проценты, суммы — точно как в PPTX v4.
   - **Тексты:** все формулировки, подписи, комментарии — как в PPTX v4.
   - **Графики/диаграммы:** данные в интерактивных графиках пересчитай под значения из PPTX v4.
   - **Таблицы:** все ячейки — до значений PPTX v4.

### Этап 3 — Улучшения дизайна (если нужно)

Если новая структура v4 требует визуальных доработок:
- Предложи конкретные улучшения.
- Реализуй после одобрения.

Допустимые улучшения:
- Новые типы визуализаций для новых данных.
- Улучшенные анимации для новых элементов.
- Адаптация layout под изменившуюся структуру слайдов.

### Этап 4 — Финальная сверка

Покажи итоговую таблицу:

| Показатель | PPTX v4 | HTML (обновл.) | Excel (справочно) |
|---|---|---|---|
| ... | ... | ... | ... |

Все значения PPTX v4 и HTML должны совпадать. Расхождения с Excel — отметить и объяснить.

## Требования к HTML (сохранить/улучшить)

- Формат 16:9, полноэкранный
- Тёмный кинематографический стиль (deep navy #0A1628 + золотые акценты #D4A843, #F5C563)
- Анимации переходов между слайдами (fade/slide)
- Анимации появления элементов (fade-in, slide-up)
- Интерактивные графики с hover/tooltip (показ точных значений)
- Навигация: клавиши ←/→, свайпы, клик
- Прогресс-бар
- Анимированные pie chart и timeline
- Если в v4 появились новые элементы — создать для них интерактивные компоненты того же уровня качества

---

## Правила

1. **PPTX v4 — единственный источник правды.** При конфликте между файлами побеждает PPTX v4.
2. **Excel НЕ трогать.** Файл «Холдинг Кино.xlsx» используется ТОЛЬКО для кросс-проверки, никаких изменений.
3. **Каждое изменение согласовывай.** Не вноси правки в HTML без одобрения пользователя.
4. **Сначала аудит — потом правки.** Всегда показывай таблицы расхождений до начала изменений.
5. **Не ломай работающее.** Если элемент HTML корректен и совпадает с v4 — не трогай его.
6. **Дизайн можно улучшить**, но только после согласования и только если это обосновано новой структурой.
