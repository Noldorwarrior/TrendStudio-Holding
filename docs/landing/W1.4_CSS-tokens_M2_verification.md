## 5. Механизм №4 — проверка границ (WCAG 2.1 contrast)

Все контрасты вычислены по WCAG 2.1 (relative luminance via sRGB→linear + `(L1+0.05)/(L2+0.05)`). Значения пересчитываются тестом; `min_ratio` в `tokens.json` — гарантированный порог.

| text / surface | **14.55** | 7.0 | AAA | PASS |

### 6.1 Known issue в валидаторе W1.2 (не блокирует W1.4)

`python3 scripts/invariants_check.py --wave W1` выдаёт `INV-06 FAIL` с 6 совпадениями:

| Файл:строка | Контекст | Тип |
|---|---|---|
| `src/landing/core/orchestrator.js:15` | JSDoc `*   3) sessionStorage['ts.state'] (НЕ localStorage — INV-06)` | доказательство запрета в документации |
| `src/landing/core/orchestrator.js:19` | JSDoc `*   — localStorage` (список запретов) | документация |
| `src/landing/core/orchestrator.js:197` | JSDoc `*  БЕЗ localStorage (INV-06)` | документация |
| `src/landing/core/__tests__/orchestrator.test.js:163` | Имя теста `'setState: сохраняется в sessionStorage (не localStorage)'` | негативная проверка |
| `src/landing/styles/__tests__/tokens.test.js:10` | JSDoc `* 5. INV-06: в CSS нет localStorage` | документация |
| `src/landing/styles/__tests__/tokens.test.js:225` | `assert.equal(/\blocalStorage\b/.test(css), false);` | проверка ОТСУТСТВИЯ |

Все 6 — false-positive. Регулярка в `invariants_check.py` ищет слово `localStorage` по всему файлу без strip комментариев и строковых литералов. Рантайм-код не содержит ни одного обращения к `localStorage` (каскад `query > hash > sessionStorage`, W1.3 подтверждено).

**Решение:** заведена задача **#31** (`W1.2-hotfix: INV-06 comment/string-strip в invariants_check.py`). Hotfix добавит обработку `//`, `/* */`, одинарных/двойных/обратных кавычек перед regex-поиском. Срок — до W1-closure П5 «Максимум» (#20).

## 10. Границы верификации

М2 «Расширенная» проверяет:
- Точный перенос из canon (№1)
- Выполнение всех пунктов запроса (№2)
- Сверку сумм и счётчиков (№3)
- Проверку границ — WCAG AA/AAA (№4)

**НЕ проверяет** (предмет П5 «Максимум» на W1-closure):
- Cross-wave согласованность с будущими W2 (CSS layout) и W5 (компоненты)
- Хронологию (когда токены могут меняться в ходе релиза)
- Парадоксы и скрытые допущения (например: fluid-clamp на h1 и accessibility при системном zoom)
- Моделирование аудитории (как LP-investor воспринимает палитру)
- Визуальное соответствие mock-ups (предмет отдельной стадии design-review)
- Проверку реальных шрифтов (Cormorant Garamond / Inter) — загрузка настраивается в HTML (W1.5)
