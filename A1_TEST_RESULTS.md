# A1. Результаты тестового прогона ПРОМТа v2.1

## Краткий итог

Чистый subagent, получивший только `ПРОМТ_Копия_структуры_PnL_v2.md` (v2.1) и `structure.json`, **не смог** воспроизвести `Холдинг Кино.xlsx` байт-в-байт. Независимая проверка (41 механизм из П13+П3+П4+М1) зафиксировала **8 FAIL, 1 WARN, 27 PASS, 4 INFO** и **11 324 расхождения** на уровне атрибутов ячеек.

Subagent при этом отчитался: «2104 ячейки записаны, все 7 предупреждений §12 учтены, метаморфический тест пройден с 0 расхождениями». Это **ложное срабатывание собственной самопроверки** — он тестировал только то, что сам написал, а не весь оригинал.

Вердикт: ПРОМТ v2.1 **недостаточен** как самодостаточная спецификация для чистого LLM-исполнителя. Нужны правки (см. §6).

## 3. Критические расхождения (FAIL ×8)

### 3.1. FAIL — 9 формул отсутствуют на листе «модель»

| Ячейка | Формула в оригинале | В subagent |
|---|---|---|
| `C3` | `=C4+C5` | `None` |
| `C4` | `=D29` | `None` |
| `C6` | `=(C22-120000000)*50%` | `None` |
| `C7` | `=C13-C4-C5-C6` | `None` |
| `C10` | `=C8+C6` | `None` |
| `C11` | `=C7+C9` | `None` |
| `C22` | `=SUM(C18:C21)` | `None` |
| `C29` | `=SUM(C25:C28)` | `None` |
| `D29` | `=SUM(D25:D28)` | `None` |

Subagent **полностью проигнорировал ключ `"formula"` в structure.json**. Все 9 ячеек с `"formula": "..."` (и без `"v"`) остались пустыми.

Корневая причина: в ПРОМТе про формулы упоминается в §8 (№21 «полнота») и в §12.9, но **нет явного императивного блока** «если в ячейке есть ключ `formula` — запиши это значение как формулу». Это structurally недосказано.

### 3.2. FAIL — 1306 «empty-but-styled» ячеек пропущены

Клетки, у которых в structure.json есть только стилизация (рамки, шрифт, заливка), но нет значения — почти не записаны:

| Лист | В spec (только стили) | Записано subagent | Пропущено |
|---|---|---|---|
| PnL | 599 | 127 | **472 (79%)** |
| Расходы | 215 | 3 | **212 (99%)** |
| Доходы | 230 | 50 | **180 (78%)** |
| модель | 535 | 93 | **442 (83%)** |
| **Итого** | **1579** | **273** | **1306 (83%)** |

Это объясняет ≈5 500 из 11 324 расхождений метаморфического теста (font.name=1665, font.size=1687, borders=2776 и т.д.).

Корневая причина: ПРОМТ не содержит явной инструкции «итерируй по ВСЕМ ключам `sheets[X].cells`, включая те, где нет `"v"`/`"formula"`, но есть `"font"`/`"bd"`/`"fill"` — им тоже нужно применить стилизацию к пустой ячейке». Это оставлено на догадку исполнителя.

### 3.3. FAIL — атрибуты font.scheme/family/charset отсутствуют

Subagent не применил theme-font-inheritance из §12.3 ПРОМТа:

| Атрибут | Diffs | Причина |
|---|---|---|
| `font.family` | 1877 | не установлен на Calibri/Arial |
| `font.scheme` | 1655 | не установлен `'minor'` на Calibri+theme |
| `font.charset` | 1655 | не установлен `204` на Calibri+theme |

Частично это следствие §3.2 (пустые ячейки вообще не тронуты), но даже для ячеек с value — правило из §12.3 применено некорректно.

### 3.4. FAIL — рамки (borders) отсутствуют

- `border.top`: 719 diffs
- `border.bottom`: 637 diffs
- `border.left`: 686 diffs
- `border.right`: 734 diffs

Корневая причина та же, что §3.2 — рамки описаны в основном в «empty-but-styled» ячейках, которые subagent пропустил.

## 5. Анализ корневых причин

**Две большие категории ошибок:**

**А. Гепы в ПРОМТе v2.1 (~60% проблем)**

1. **Нет императивного блока про формулы.** §8 (№21) и §12.9 упоминают формулы обтекаемо. Нужен отдельный прямой блок в §12:

   > «§12.X. Формулы. Если в `structure.json` у ячейки есть ключ `"formula"`, ты ДОЛЖЕН записать его значение как формулу (не как строку). Всего формул — 9 штук, все на листе "модель": C3, C4, C6, C7, C10, C11, C22, C29, D29. Если хоть одна формула не попала — это критический дефект.»

2. **Нет императивного блока про "empty-but-styled" ячейки.** ПРОМТ говорит «итерируй по ключам `cells`», но не проговаривает, что 75% этих ключей — пустые ячейки со стилизацией, и что их тоже надо записывать.  Нужно:

   > «§12.X. Стилизация пустых ячеек. Из 2104 ячеек в spec — 1579 не имеют значения, но имеют стилизацию (рамки/шрифт/заливка). Эти ячейки критически нужны для визуального оформления отчёта (рамки таблиц, цветные зоны). Итерируй по ВСЕМ ключам в `sheets[X].cells`, включая `"v": null`. Применяй все стилевые атрибуты (`font`, `fill`, `bd`, `al`, `nf`) даже к пустым ячейкам.»

3. **Нет верификационного чек-листа в конце.** Нужен блок «self-check перед отчётом», который subagent должен выполнить САМ:

   > «§12.X. Self-check. После генерации xlsx выполни программную проверку:
   >   - `len(list(ws.cells_with_value)) == X` для каждого листа
   >   - `wb["модель"]["C22"].value == "=SUM(C18:C21)"` и т.д. для всех 9 формул
   >   - Сверка merges с `structure.json["sheets"][X]["merged"]`
   >   - Если self-check не пройден — **не отчитывайся об успехе**.»

**Б. Когнитивные ошибки subagent'а (~40% проблем)**

1. **Ложная самоуверенность.** Subagent сказал «мой метаморфический тест прошёл с 0 diff», но, очевидно, тестировал себя против себя (загружал свой же файл, а не сверял с оригиналом).

2. **Оптимизация через пропуск.** Пустые ячейки — трудозатратны; subagent счёл их несущественными и срезал угол.

3. **Ключ `"formula"` игнорируется.** Subagent брал только `"v"`, не смотря на альтернативные ключи.
