## Методология

### Принцип работы
1. **Этапный режим:** завершить блок → показать промежуточные результаты → получить подтверждение → следующий блок
2. **Скриптовая верификация:** для каждой критической проверки — написать Python-скрипт, запустить, зафиксировать stdout/stderr
3. **Двойной расчёт:** ключевые метрики (IRR, EBITDA, NDP, BS balance, Cash chain) пересчитывать независимо от модели
4. **Не доверять кэшам:** формулы пересчитывать программно, кэшированные значения в xlsx — только для сравнения
5. **Оба файла:** Internal и Public проверять параллельно, расхождения = находка

- `extra="forbid"` на всех моделях — нет ли утечек

## Правила выполнения

1. **Этапный режим:** блок за блоком. Завершил A → показал результаты → подтверждение → B → ... → D
2. **Скрипты сначала:** перед ручным анализом запустить автоматические скрипты, потом дополнить ручными находками
3. **Не доверять кэшам:** пересчитывать формулы программно
4. **Оба файла:** Internal + Public параллельно; расхождение = находка
5. **Read-only:** ничего не модифицировать в проверяемых файлах
6. **Критические баги:** сообщать немедленно, не ждать конца блока
7. **Каждая находка — с доказательством:** файл, лист, ячейка/строка, конкретные числа
8. **v1.0 — только diff:** полный аудит v3; для v1.0 — регрессионное сравнение
9. **Скрипты сохранять:** все 10 скриптов сохранить в `pipeline/scripts/audit/` для повторного использования
