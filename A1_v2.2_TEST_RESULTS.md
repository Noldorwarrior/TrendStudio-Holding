## 2. Что не закрылось — новый геп «Partial font replacement»

### 2.1. Находка

В `structure.json` многие ячейки описаны с **неполной** font-спецификацией, например:

```json
"PnL!B2": {"font": {"color": {"theme": 1}}}
```

Subagent v2.2 создал Font-объект так:

```python
cell.font = Font(color=Color(theme=1))
```

Это **заменило** унаследованный дефолтный шрифт (Calibri 12pt theme minor) на объект с ТОЛЬКО color-атрибутом и всеми остальными = None. В результате 1655 ячеек оказались с `font.name=None`, `font.size=None`, `font.scheme=None`.

### 2.2. Масштаб

Ячеек с потерянным `font.name`:

| Лист | Потеряно |
|---|---|
| PnL | 633 |
| Расходы | 320 |
| Доходы | 222 |
| модель | 480 |
| **Итого** | **1 655** |

Это даёт почти одинаковое количество diff по всем 5 theme-атрибутам: `font.name=1655, font.size=1677, font.scheme=1665, font.family=1887, font.charset=1665`.

### 2.3. Как должно было быть

Правильный подход: **merge** override-атрибутов из `structure.json` с дефолтным шрифтом листа, а не замещать:

```python
def build_font(font_spec, default_font=None):
    default_font = default_font or {
        "name": "Calibri", "size": 12, "family": 2,
        "scheme": "minor", "charset": 204,
    }
    # Смержить default_font с font_spec
    kwargs = dict(default_font)
    if font_spec:
        for k, v in font_spec.items():
            kwargs[k] = v
    # Обработать color
    color = font_spec.get("color") if font_spec else None
    if color is not None:
        kwargs["color"] = build_color(color)
        # Если color - RGB, разорвать theme-inheritance (из §12.3)
        if isinstance(color, str):
            kwargs["family"] = None
            kwargs["scheme"] = None
            kwargs["charset"] = None
    return Font(**kwargs)
```

### 2.4. Почему этого нет в ПРОМТе v2.2

§12.2 описывает openpyxl workaround для `wb._fonts[0]` — это фиксит дефолт для ПУСТЫХ ячеек без каких-либо стилей. Но если ячейка имеет `"font": {"color": {...}}` — subagent применяет `cell.font = Font(...)`, что создаёт новый fontId, не наследующий дефолт.

§12.3 описывает theme-font inheritance для RGB vs theme colors — но предполагает, что name и size уже правильные.

**Геп:** нет явного блока «partial font spec должен мержиться с дефолтом, не заменять». Это третий критический класс гепа в ПРОМТе.

## 5. Рекомендации для ПРОМТа v2.3

### 5.1. Новый §12.14 — Partial font merge

```markdown
### 12.14. Partial font spec — MERGE with default, don't REPLACE

Многие ячейки в `structure.json` имеют НЕПОЛНУЮ font-спецификацию:
- `{"font": {"color": {"theme": 1}}}` — только цвет
- `{"font": {"b": true}}` — только bold
- `{"font": {"name": "Arial"}}` — только имя

Ты **ОБЯЗАН** мержить эти override-атрибуты с дефолтным шрифтом листа
(Calibri 12pt theme minor), а НЕ заменять шрифт целиком.

**ПРАВИЛЬНО:**
```python
def build_font(spec):
    kwargs = {
        "name": "Calibri", "size": 12, "family": 2,
        "scheme": "minor", "charset": 204,
    }
    if spec:
        for key in ("name", "size", "family", "scheme", "charset"):
            if key in spec:
                kwargs[key] = spec[key]
        if "b" in spec: kwargs["bold"] = spec["b"]
        if "i" in spec: kwargs["italic"] = spec["i"]
        if "color" in spec:
            kwargs["color"] = build_color(spec["color"])
            # Из §12.3: RGB разрывает theme-inheritance
            if isinstance(spec["color"], str):
                kwargs["family"] = None
                kwargs["scheme"] = None
                kwargs["charset"] = None
    return Font(**kwargs)
```

**НЕПРАВИЛЬНО (v2.2 subagent повторил эту ошибку):**
```python
cell.font = Font(color=Color(theme=1))  # ← теряет name, size, family, scheme
```

**Контроль:** после генерации проверить, что cells с `font.name=None` <= 10.
Если больше — значит не смержил, а заместил.
```
