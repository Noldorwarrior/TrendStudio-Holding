# Промт: Безжалостный аудит финансовой модели кинохолдинга v4.0

## Структура

```
промт_аудит_v4/
├── 00_CORE.md      ← Ядро: роль, принципы, тон, данные, формат DOCX, self-check
├── 01_P0.md        ← Модуль P0: блоки 1, 4, 5 (стоп-факторы)
├── 02_P1.md        ← Модуль P1: блоки 2, 3, 7 (бенчмаркинг, доходы, стресс-тест)
├── 03_P2.md        ← Модуль P2: блоки 6, 8–11, адвокат дьявола
├── 04_GOLDEN.md    ← Эталонные расчёты для самопроверки
└── README.md       ← Этот файл
```

## Как использовать

### Вариант 1: Последовательная загрузка (рекомендуется)

Подавать в Claude (Cowork/Code) файлы по одному в четырёх сообщениях:

**Сообщение 1:**
> Прочитай файлы `00_CORE.md` и `04_GOLDEN.md`. Это системная инструкция и эталонные расчёты для аудита финансовой модели кинохолдинга. Подтверди, что понял роль, данные и формат вывода.

**Сообщение 2:**
> Теперь прочитай `01_P0.md` и выполни блоки 1, 4, 5. Выдай промежуточный вердикт P0. Результат запиши — он войдёт в финальный DOCX.

**Сообщение 3:**
> Прочитай `02_P1.md`. Выполни блоки 2, 3, 7 (обязателен WebSearch для блока 2). Выдай промежуточный вердикт P1.

**Сообщение 4:**
> Прочитай `03_P2.md`. Выполни блоки 6, 8–11 и «Адвокат дьявола» (WebSearch для блоков 9, 11). Выдай финальный вердикт. Собери весь отчёт в DOCX по структуре из `00_CORE.md` и сохрани в папку.

### Вариант 2: Всё в одном сообщении

> Прочитай все файлы из папки `промт_аудит_v4/` и выполни полный аудит финансовой модели кинохолдинга. Следуй порядку: P0 → P1 → P2. Используй WebSearch для блоков 2, 9, 11. Сверяй расчёты с `04_GOLDEN.md`. Выводи промежуточные чекпоинты. Финал — DOCX-отчёт на 25–35 страниц.

## Что нового в v4.0 (vs v3.1)

| Улучшение | Описание |
|-----------|----------|
| Модульная структура | Ядро + 3 модуля по приоритетам (вместо одного монолитного промта) |
| Промежуточные чекпоинты | После P0, P1, P2 — промежуточные вердикты с возможностью досрочного стопа |
| Эталонные расчёты | Файл `04_GOLDEN.md` с предвычисленными формулами для самопроверки |
| Визуализации | 5 диаграмм: waterfall, матрица рисков 2×2, timeline, radar-chart, сценарный график |
| Адвокат дьявола | Блок 12: аргументы В ПОЛЬЗУ инвестиции + контр-опровержения (защита от confirmation bias) |
| Claude-специфичные инструкции | XML-теги, ссылки на WebSearch/Bash/python-docx, скилл docx |
| Усиленный few-shot | 3 примера тона (вместо 2), включая пример для структуры сделки |
| Расширенные anti-patterns | Добавлены общие выводы без формул |
| Fallback при расхождении с эталоном | Пересчёт дважды + указание обоих значений |

## Ограничения

- Промт одноразовый (данные ТрендСтудио зашиты)
- Требует Claude с доступом к WebSearch, Bash, python-docx
- Эталонные расчёты — ориентиры, не абсолютная истина (допуск 5%)
- Визуализации требуют matplotlib в среде исполнения
