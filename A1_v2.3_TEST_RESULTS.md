### 3.1. Разбивка по типам

### 3.2. Root cause #1 — формат ключей `b`/`i` (47 diff)

В `structure.json` используются сокращения:

```json
"font": {"name": "Helvetica Neue", "size": 16.0, "b": true, "color": "FF0F1115"}
```

Ключ `b` — shorthand для `bold`, ключ `i` — для `italic`. Это задокументировано в `meta.cell_format_legend`:

> `"font": "отличия от default_font листа (name, size, b, i, color)"`

Но **шаблон `build_font()` в §12.14 ПРОМТа** обрабатывает только `bold`/`italic`:

```python
# ПРОМТ v2.3 §12.14:
if "bold" in font_spec:
    kwargs["bold"] = font_spec["bold"]
if "italic" in font_spec:
    kwargs["italic"] = font_spec["italic"]
```

Subagent следовал §12.14 дословно — ключ `b` не распознан → bold не применён → 44 diff по `font.bold` + 3 по `font.italic`.

**В spec:** 44 ячейки с `"b": true`, 3 — с `"i": true`. Идеальное совпадение с числом diff.

### 3.3. Root cause #2 — color как строка (229 diff)

В `structure.json` цвет в `font` хранится в **трёх форматах**:

```json
"font": {"color": {"theme": 1}}                // 1. объект theme
"font": {"color": {"rgb": "FF0F1115"}}         // 2. объект rgb (встречается реже)
"font": {"color": "FFF9FAFB"}                  // 3. просто строка-hex
```

§12.14 обрабатывает только **два** варианта: dict с `theme` и dict с `rgb`. Строка-hex не обрабатывается.

**В spec:** 223 ячейки с color-как-string. Subagent оставил им default color (theme=1), отсюда 229 diff (разница с 223 — 6 ячеек где color ещё и конвертируется в RGB другого типа).

### 3.4. Root cause #3 — fill.fg как строка (86 diff)

Fill хранится в двух формах:

```json
"fill": {"pattern": "solid", "fg": "FF151517", "bg": "00000000"}      // 1. строки
"fill": {"pattern": "solid", "fg": {"theme": 4, "tint": -0.5}}        // 2. объекты
```

Subagent обработал только dict-форму. 23 ячейки с fill.fg-как-string не получили заливку → 43 diff по `fill.pattern` + 43 по `fill.fgColor` (86 суммарно — ≈2 атрибута на ячейку, сходится).

Это самый визуально разрушительный остаточный баг — **тёмные шапки разделов** на листах `Расходы` и `Доходы` (заливка `#151517`, шрифт `#F9FAFB`) вообще не отрисованы.

### 3.5. Root cause #4 — non-Calibri шрифты (30 diff)

Лист `модель` использует `Helvetica Neue 18pt bold` для главного заголовка, `Helvetica Neue 16pt bold` для шапок блоков. По правилу §12.3:

| Шрифт | family | scheme | charset |
|---|---|---|---|
| Calibri+theme | 2 | "minor" | 204 |
| Helvetica Neue | 2 | None | None |

Subagent применил Calibri-правила ко всем шрифтам без учёта non-Calibri ветки. Это 10 ячеек × 3 атрибута = 30 diff.

Код §12.14 содержит частичный хэндлинг non-Calibri:

```python
if kwargs["name"] != "Calibri":
    kwargs["scheme"] = None
    kwargs["charset"] = None
```

Но subagent, вероятно, применил эту логику **после** полного merge, и из-за порядка операций `family` остался как 2 (должен был тоже обнулиться? — нет, по §12.3 `family=2` для Helvetica Neue. Здесь проблема в `scheme`/`charset`). Посмотрим точнее: diff `font.family: 10` говорит, что orig=None, sub=2 — т.е. наоборот, subagent **применил** family=2, а в оригинале None. Значит оригинал для Helvetica Neue имеет `family=None`, а не 2.

**Это противоречит таблице в §12.3**, где написано «Helvetica Neue → family=2». Нужно перепроверить и поправить §12.3.

## 5. Корневая причина остатка — неполнота легенды в §12.14

**Это не vs сам subagent. Это vs ПРОМТ.**

§12.14 предоставляет `build_font()`-шаблон, но он не покрывает 4 формы записи, которые действительно встречаются в `structure.json`:

1. `font.b` вместо `font.bold` (44 случая в spec)
2. `font.i` вместо `font.italic` (3 случая)
3. `font.color` как string `"FFF9FAFB"` вместо dict (223 случая)
4. `fill.fg` как string `"FF151517"` вместо dict (23 случая)

Плюс:

5. Таблица §12.3 говорит «Helvetica Neue → family=2», а в оригинале для Helvetica Neue стоит `family=None` (10 случаев).

**Легенда `meta.cell_format_legend` сама по себе неполная:**

```json
"font": "отличия от default_font листа (name, size, b, i, color)"
```

Не сказано, что `color` может быть string. Не сказано, что есть ключи `u` (underline), `strike`, `vertAlign`. Не сказано, что fill.fg может быть string.

### 6.1. Расширить `build_font()` в §12.14

```python
def build_font(font_spec):
    kwargs = dict(DEFAULT_FONT_KWARGS)
    if not font_spec:
        return Font(**kwargs)

    if "name" in font_spec: kwargs["name"] = font_spec["name"]
    if "size" in font_spec: kwargs["size"] = font_spec["size"]

    # Bold/italic/underline/strike — поддержка shorthand
    if "bold" in font_spec or "b" in font_spec:
        kwargs["bold"] = font_spec.get("bold", font_spec.get("b"))
    if "italic" in font_spec or "i" in font_spec:
        kwargs["italic"] = font_spec.get("italic", font_spec.get("i"))
    if "u" in font_spec: kwargs["underline"] = font_spec["u"]
    if "strike" in font_spec: kwargs["strike"] = font_spec["strike"]

    # Color: dict ({theme|rgb|indexed|auto}) ИЛИ string ("RRGGBB"/"AARRGGBB")
    if "color" in font_spec:
        c = font_spec["color"]
        if isinstance(c, str):
            # 6- или 8-символьный hex → RGB
            kwargs["color"] = Color(rgb=c)
            kwargs["family"] = None
            kwargs["scheme"] = None
            kwargs["charset"] = None
        elif isinstance(c, dict):
            if "theme" in c:
                kwargs["color"] = Color(theme=c["theme"], tint=c.get("tint", 0.0))
            elif "rgb" in c:
                kwargs["color"] = Color(rgb=c["rgb"])
                kwargs["family"] = None
                kwargs["scheme"] = None
                kwargs["charset"] = None
            elif "indexed" in c:
                kwargs["color"] = Color(indexed=c["indexed"])
            elif c.get("auto"):
                kwargs["color"] = Color(auto=True)

    # Non-Calibri
    if kwargs["name"] and kwargs["name"] != "Calibri":
        kwargs["scheme"] = None
        kwargs["charset"] = None
        if kwargs["name"] in ("Helvetica Neue", "quote-cjk-patch"):
            kwargs["family"] = None
        # Arial остаётся family=2

    return Font(**kwargs)
```

### 6.2. Добавить `build_fill()` в §12.14

```python
def build_fill(fill_spec):
    if not fill_spec:
        return None
    from openpyxl.styles import PatternFill, Color
    pattern = fill_spec.get("pattern", "solid")

    def _to_color(v):
        if isinstance(v, str):
            return Color(rgb=v)
        if isinstance(v, dict):
            if "theme" in v:
                return Color(theme=v["theme"], tint=v.get("tint", 0.0))
            if "rgb" in v:
                return Color(rgb=v["rgb"])
            if "indexed" in v:
                return Color(indexed=v["indexed"])
        return None

    fg = _to_color(fill_spec.get("fg"))
    bg = _to_color(fill_spec.get("bg"))
    return PatternFill(patternType=pattern, fgColor=fg, bgColor=bg)
```
