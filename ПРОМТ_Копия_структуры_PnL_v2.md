##### Текст ячейки `A5` (merge `A5:A43`)

```
Примем:
	• Производственные инвестиции: 312,5 млн руб. в квартал в течение 4 кварталов (1-4).
	• Операционные расходы: 22,12625 млн руб. в квартал (постоянно, с 1 по 12 квартал).
	• Доходы начинаются с 5 квартала и идут по 12 квартал включительно (8 кварталов). Обозначим X – ежеквартальный доход (одинаковый для упрощения, но можно и неравномерный; для расчёта среднего достаточно).
Тогда кумулятивный чистый поток на конец 8 кварталов (2 года) должен быть равен 0.
На конец 8 кварталов:
	• Суммарные инвестиции в производство: 4 * 312,5 = 1250 млн руб.
	• Суммарные операционные расходы за 8 кварталов: 8 * 22,12625 = 177,01 млн руб.
	• Суммарные доходы за 5-8 кварталы: 4 * X (доходы поступают только 4 квартала к концу 2-го года).
Чистый поток = Доходы – Расходы (производственные + операционные) = 4X – (1250 + 177,01) = 0.
=> 4X = 1427,01 => X = 356,7525 млн руб. в квартал.
Теперь проверим, что на конец 3-го года (12 кварталов) кумулятивный поток станет +2000 млн руб. (сверх окупаемости). К концу 12 кварталов:
	• Производственные инвестиции: 1250 млн руб. (уже учтены)
	• Операционные расходы за 12 кварталов: 12 * 22,12625 = 265,515 млн руб.
	• Доходы за 5-12 кварталы: 8 * X = 8 * 356,7525 = 2854,02 млн руб.
Чистый поток = 2854,02 – (1250 + 265,515) = 2854,02 – 1515,515 = 1338,505 млн руб.
Это не 2000, а 1338,5. Значит, если окупаемость ровно в конце 8 квартала, то к концу 12 квартала накопленный чистый поток будет около 1338,5 млн руб. Нам же нужно 2000 млн руб. на конец 12 квартала.
Значит, требуется либо увеличить ежеквартальные доходы (чтобы и к 8 кварталу был ноль, и к 12-му больше), либо перераспределить доходы (например, сделать их выше в последние кварталы). В модели с равномерными доходами можно подобрать X так, чтобы выполнялись оба условия одновременно? Невозможно, так как два условия: 4X – (1250+8*22,12625)=0 и 8X – (1250+12*22,12625)=2000. Эти уравнения несовместны. Поэтому нужно либо изменить форму доходов (например, не равномерные, а с нарастанием), либо пересмотреть условие окупаемости за 2 года как момент, когда кумулятивный поток становится положительным, а не точно ноль в конце 8 квартала. Однако в условии сказано «выйти на окупаемость за 2 года и на 3 год заработать еще 2 миллиарда рублей». Это может означать, что окупаемость (возврат инвестиций) достигается к концу 2-го года, а за третий год чистая прибыль составляет 2 млрд руб. (т.е. за период с 9 по 12 квартал чистый доход = 2000 млн руб.). Тогда это возможно.
Примем вторую интерпретацию: к концу 8 кварталов кумулятивный поток = 0 (окупаемость). За 3-й год (кварталы 9-12) чистый поток должен составить 2000 млн руб. То есть за 4 квартала 3-го года нужно получить 2000 млн руб. чистого дохода после вычета операционных расходов. Операционные расходы в год – 88,505 млн руб., за 4 квартала – 88,505 млн руб. (или ровно за год). Значит, за 3-й год доходы должны быть: 2000 + 88,505 = 2088,505 млн руб. Если доходы поступают равномерно по кварталам 3-го года, то ежеквартально это 522,12625 млн руб.
Но при этом в первые два года доходы были меньше, чтобы обеспечить окупаемость к концу 2-го года. Рассчитаем отдельно для 1-го и 2-го годов.
Сделаем более общий расчёт: пусть ежеквартальные доходы не обязательно равномерны, но для простоты разделим на два периода: первые два года (кварталы 5-8) – доход Y в квартал; третий год (кварталы 9-12) – доход Z в квартал. Нам нужно:
	1. К концу 8 квартала кумулятивный поток = 0.
	2. Чистый поток за 3-й год (9-12 кварталы) = 2000 млн руб. (чистый, т.е. доходы минус операционные расходы за эти 4 квартала).
Операционные расходы за 8 кварталов: 8 * 22,12625 = 177,01 млн руб.
Производственные инвестиции за 4 квартала: 1250 млн руб.
Доходы за кварталы 5-8: 4Y.
Уравнение окупаемости к концу 8 квартала:
4Y – (1250 + 177,01) = 0 => 4Y = 1427,01 => Y = 356,7525 млн руб.
Третий год:
Операционные расходы за 4 квартала: 4 * 22,12625 = 88,505 млн руб.
Доходы за 9-12 кварталы: 4Z.
Чистый поток за 3-й год: 4Z – 88,505 = 2000 => 4Z = 2088,505 => Z = 522,12625 млн руб.
Таким образом, чтобы окупиться за 2 года и заработать за 3-й год 2 млрд руб. чистыми, необходимо:
	• В течение первого года после производства (5-8 кварталы) получать ежеквартальные доходы около 356,75 млн руб.
	• В течение третьего года (9-12 кварталы) – ежеквартальные доходы около 522,13 млн руб.
Суммарные доходы за 3 года (8 кварталов поступлений) составят:
4*356,75 + 4*522,13 = 1427,01 + 2088,52 = 3515,53 млн руб.
Суммарные расходы (производство + операционные за 3 года): 1250 + 265,515 = 1515,515 млн руб.
Чистый результат: 3515,53 – 1515,515 = 2000,015 млн руб.
```

## 9. СОПРОВОДИТЕЛЬНЫЙ ФАЙЛ `structure.json`

Наряду с этим промтом исполнителю передаётся файл `structure.json` — машиночитаемая спецификация всех ячеек. Формат:

```json
{
  "sheets": [
    {
      "name": "PnL",
      "column_widths": {"A": 191.88, "B": 12.70, "C": 13.81},
      "row_heights": {"1": 40.5, "5": 27.75, "44": 30, "74": 30, "87": 2.25},
      "merged": ["A1:A4", "A5:A43", "A57:A72", "A87:H95"],
      "cells": {
        "A44": {
          "v": "Квартал",
          "font": {"name": "Calibri", "size": 12},
          "al": {"h": "center", "wrap": true},
          "bd": "all-thin"
        }
      }
    }
  ]
}
```

**Ключи:**
- `v` — значение ячейки (для `[TBD]` — строка `"[TBD]"`; для формул — строка с `=...`).
- `nf` — число-формат (опционально; если отсутствует — `General`).
- `font` — `{name, size, b (bold), i (italic), color (hex RRGGBB без альфы)}`.
- `fill` — цвет заливки (hex) или `null`.
- `al` — `{h (horizontal), v (vertical), wrap}`.
- `bd` — `"all-thin"` если все 4 стороны тонкие, иначе `{top, bottom, left, right: "thin|medium|..."}`.

Cell-maps в JSON-файле пополняются в итерациях 2–4 параллельно с этим markdown-файлом.

## 12. КРИТИЧЕСКИЕ ОПОВЕЩЕНИЯ ДЛЯ ИСПОЛНИТЕЛЯ (результаты аудита)

### 12.1. Default font — не Calibri 11, а Calibri 12

Оригинал использует **Calibri 12pt** как шрифт по умолчанию, а не Excel-дефолт Calibri 11pt. Это относится ко всем 4 листам.

### 12.2. Openpyxl — ловушка сериализации default font

Если исполнитель использует `openpyxl` для генерации файла, простое создание `Workbook()` даёт дефолтный шрифт Calibri 11. Прямое задание `Font(size=12)` на ячейках **не сохраняется при save/load** для пустых ячеек с theme-color шрифтом: openpyxl дедуплицирует стили и теряет размер.

**Обязательный workaround** (до создания листов и ячеек):

```python
from openpyxl import Workbook
from openpyxl.styles import Font, Color

wb = Workbook()
wb.remove(wb.active)

# ОБЯЗАТЕЛЬНО: заменить wb._fonts[0] ДО создания любых ячеек.
# Параметры Calibri theme-font из оригинала.
wb._fonts[0] = Font(name="Calibri", size=12, family=2,
                    scheme="minor", charset=204, color=Color(theme=1))
wb._named_styles['Normal'].font = Font(name="Calibri", size=12, family=2,
                                         scheme="minor", charset=204,
                                         color=Color(theme=1))
```

### 12.3. Font.family / scheme / charset — theme-font inheritance

Оригинал назначает шрифтам theme-font атрибуты **в зависимости от типа цвета**:

| Шрифт              | Цвет       | `family` | `scheme`  | `charset` |
|--------------------|------------|----------|-----------|-----------|
| `Calibri`          | theme      | `2`      | `"minor"` | `204`     |
| `Calibri`          | нет цвета  | `2`      | `"minor"` | `204`     |
| `Calibri`          | RGB        | `None`   | `None`    | `None`    |
| `Arial`            | любой      | `2`      | `None`    | `None`    |
| `Helvetica Neue`   | любой      | `2`      | `None`    | `None`    |
| `quote-cjk-patch`  | любой      | `None`   | `None`    | `None`    |

Правило: **theme-цвет наследует theme-font-атрибуты**, **RGB-цвет разрывает наследование** (scheme=None, family=None, charset=None). `charset=204` — это ANSI Cyrillic (Windows-1251), locale-specific для русскоязычных файлов.

### 12.4. Строки маркеров — должны остаться строками

`PnL!F82 = "-0,04 ≈ 0"`, `Доходы!D56 = "~1 515"`, `модель!B102 = "..."`, `модель!B113 = "..."`, `модель!B115 = "28-59"`, `модель!D111/D112/D114` — это **строковые** ячейки-маркеры (не числа). Не преобразовывать в числа даже если ячейка имеет числовой формат (`nf`).

### 12.5. Двойной 4.1. на листе `модель` — это НЕ опечатка

`A32` и `A34` оба содержат `"4.1."` — это осознанная нумерация оригинала. Не исправлять.

### 12.6. NBSP (U+00A0) — обязательно сохранять

В оригинале встречается 87 неразрывных пробелов (U+00A0): PnL=21, Расходы=20, Доходы=40, модель=6. Все внутри строковых значений ячеек. Не заменять на обычный пробел — это может визуально сломать форматирование чисел в исходном тексте.

### 12.7. customHeight=15.0 — строки с явной «стандартной» высотой

150 строк (89 на `PnL`, 61 на `Доходы`) имеют `customHeight=True` с высотой `15.0` — это НЕ дефолт, оригинал явно сохранил их как «кастомные стандартные». Устанавливать `row_dimensions[n].height = 15.0` для этих строк.

### 12.8. Zoom листа `модель` = 75%

Только `модель` имеет `sheet_view.zoomScale = 75`. Остальные — без установленного zoom (дефолт 100%).

### 12.9. Метаморфический тест — как проверить

После генерации файла исполнитель **обязан** выполнить метаморфический тест: прочитать свой xlsx обратно через `openpyxl` и сверить **с оригиналом** `Холдинг Кино.xlsx` ячейка-за-ячейкой. Целевое значение — **0 расхождений**.

> **⚠️ Антиловушка самооценки.** Тест должен сверять xlsx **с оригиналом** `Холдинг Кино.xlsx`. Сверка своего файла «сам с собой» (загрузил → перечитал → сравнил) **не считается** метаморфическим тестом и всегда даст 0 diff. В A1-тестах v2.1 и v2.3 этот антипаттерн привёл к ложным отчётам «0 расхождений», в то время как реальная сверка с оригиналом показала 11 324 и 392 diff соответственно.

**Обязательный эталонный код (A1-fix v2.4). Использовать без модификаций:**

```python
from openpyxl import load_workbook

def metamorphic_test(sub_path: str, orig_path: str = "Холдинг Кино.xlsx") -> int:
    """Сравнить сгенерированный xlsx с оригиналом. Вернуть количество diff.
    НЕ модифицируй логику — функция должна быть dumb comparator."""
    orig = load_workbook(orig_path, data_only=False)
    sub  = load_workbook(sub_path,  data_only=False)
    if sub.sheetnames != orig.sheetnames:
        print(f"SHEET ORDER diff: {sub.sheetnames} vs {orig.sheetnames}")
        return -1

    def col_sig(col):
        if col is None: return None
        t = getattr(col, "type", None)
        if t == "rgb":     return ("rgb", col.rgb)
        if t == "theme":   return ("theme", col.theme, round(col.tint or 0, 4))
        if t == "indexed": return ("indexed", col.indexed)
        if t == "auto":    return ("auto",)
        return None

        s = getattr(border_obj, side, None) if border_obj else None
        return s.style if s is not None else None

    details = []
    for sn in orig.sheetnames:
        o_ws, s_ws = orig[sn], sub[sn]
        for row in o_ws.iter_rows():
            for c in row:
                sc = s_ws[c.coordinate]
                checks = [
                    ("value",        c.value, sc.value),
                    ("font.name",    c.font.name, sc.font.name),
                    ("font.size",    c.font.size or 0, sc.font.size or 0),
                    ("font.bold",    c.font.bold, sc.font.bold),
                    ("font.italic",  c.font.italic, sc.font.italic),
                    ("font.scheme",  c.font.scheme, sc.font.scheme),
                    ("font.family",  c.font.family, sc.font.family),
                    ("font.charset", c.font.charset, sc.font.charset),
                    ("font.color",   col_sig(c.font.color), col_sig(sc.font.color)),
                    ("fill.pattern", c.fill.patternType or None, sc.fill.patternType or None),
                    ("fill.fg",      col_sig(c.fill.fgColor), col_sig(sc.fill.fgColor)),
                    ("al.h",         c.alignment.horizontal, sc.alignment.horizontal),
                    ("al.v",         c.alignment.vertical, sc.alignment.vertical),
                    ("al.wrap",      c.alignment.wrap_text, sc.alignment.wrap_text),
                    ("bd.top",       bside(c.border,"top"), bside(sc.border,"top")),
                    ("bd.bottom",    bside(c.border,"bottom"), bside(sc.border,"bottom")),
                    ("bd.left",      bside(c.border,"left"), bside(sc.border,"left")),
                    ("bd.right",     bside(c.border,"right"), bside(sc.border,"right")),

    for d in details:
        print(" ", d)
```

#### 12.14.1. Все формы записи `font` в `structure.json`

Легенда `meta.cell_format_legend` описывает ключи шрифта как `name, size, b, i, color`. Реально встречаются:

| Ключ | Формы | Пример |
|---|---|---|
| `name` | string | `"Calibri"`, `"Helvetica Neue"`, `"Arial"`, `"quote-cjk-patch"` |
| `size` | число | `12.0`, `16.0`, `18.0`, `20.0` |
| `b` ИЛИ `bold` | bool | `"b": true` (shorthand, 44 ячейки в spec) |
| `i` ИЛИ `italic` | bool | `"i": true` (shorthand, 3 ячейки) |
| `u` ИЛИ `underline` | string | `"single"`, `"double"`, `"none"` |
| `strike` | bool | — |
| `color` | string ИЛИ dict | см. ниже |

**Формы `color`:**

1. **String** (hex, 6 или 8 символов, без `#`): `"FFF9FAFB"`, `"FF0F1115"`, `"0F1115"`. → openpyxl `Color(rgb="RRGGBB"/"AARRGGBB")`
2. **dict с `theme`**: `{"theme": 1}` или `{"theme": 4, "tint": -0.5}` → `Color(theme=..., tint=...)`
3. **dict с `rgb`**: `{"rgb": "FF0F1115"}` → `Color(rgb=...)`
4. **dict с `indexed`**: `{"indexed": 64}` → `Color(indexed=...)`
5. **dict с `auto`**: `{"auto": true}` → `Color(auto=True)`

**Правило theme-font inheritance (§12.3):** если цвет — RGB (string или `{"rgb": ...}`), то для шрифта **Calibri** принудительно `family=None, scheme=None, charset=None`. Для theme-color наследование сохраняется: `family=2, scheme="minor", charset=204`.

---

#### 12.14.2. Все формы записи `fill` в `structure.json`

```json
"fill": { "pattern": "solid", "fg": <color>, "bg": <color> }
```

Где `<color>` — **те же 5 форм**, что у `font.color`: string или dict с theme/rgb/indexed/auto.

#### 12.14.3. Правильные функции build_font и build_fill (обязательный эталон)

```python
from openpyxl.styles import Font, Color, PatternFill

# Default для листа: Calibri 12pt theme-minor с theme=1 color
DEFAULT_FONT_KWARGS = dict(
    name="Calibri", size=12, family=2,
    scheme="minor", charset=204, color=Color(theme=1),
    bold=False, italic=False, underline="none", strike=False, vertAlign=None,
)

def _parse_color(c):
    """Принять любую из 5 форм color и вернуть (Color, is_rgb).
    is_rgb=True если это RGB (важно для theme-font inheritance)."""
    if c is None:
        return None, False
    if isinstance(c, str):
        # Hex string "RRGGBB" или "AARRGGBB"
        rgb = c if len(c) == 8 else ("FF" + c)  # добавить alpha если нужно
        return Color(rgb=rgb), True
    if isinstance(c, dict):
        if "theme" in c:
            return Color(theme=c["theme"], tint=c.get("tint", 0.0)), False
        if "rgb" in c:
            rgb = c["rgb"]
            if len(rgb) == 6:
                rgb = "FF" + rgb
            return Color(rgb=rgb), True
        if "indexed" in c:
            return Color(indexed=c["indexed"]), False
        if c.get("auto"):
            return Color(auto=True), False
    return None, False


def build_font(font_spec):
    """Merge override attributes with Calibri 12 theme-minor default."""
    kwargs = dict(DEFAULT_FONT_KWARGS)
    if not font_spec:
        return Font(**kwargs)

    if "name" in font_spec:
        kwargs["name"] = font_spec["name"]
    if "size" in font_spec:
        kwargs["size"] = font_spec["size"]

    # Bold: поддержать оба написания (shorthand "b" и полное "bold")
    if "bold" in font_spec:
        kwargs["bold"] = bool(font_spec["bold"])
    elif "b" in font_spec:
        kwargs["bold"] = bool(font_spec["b"])

    # Italic: то же самое
    if "italic" in font_spec:
        kwargs["italic"] = bool(font_spec["italic"])
    elif "i" in font_spec:
        kwargs["italic"] = bool(font_spec["i"])

    # Underline / strike
    if "underline" in font_spec:
        kwargs["underline"] = font_spec["underline"]
    elif "u" in font_spec:
        kwargs["underline"] = font_spec["u"]
    if "strike" in font_spec:
        kwargs["strike"] = bool(font_spec["strike"])

    # Color: string ИЛИ dict (5 форм)
    is_rgb_color = False
    if "color" in font_spec:
        col, is_rgb_color = _parse_color(font_spec["color"])
        if col is not None:
            kwargs["color"] = col

    # Theme-font inheritance (§12.3):
    # RGB color разрывает theme-attrs для Calibri
    if is_rgb_color and kwargs["name"] == "Calibri":
        kwargs["family"] = None
        kwargs["scheme"] = None
        kwargs["charset"] = None

    # Non-Calibri шрифты (§12.3):
    # Helvetica Neue → family=2, scheme=None, charset=None
    # Arial           → family=2, scheme=None, charset=None
    # quote-cjk-patch → family=None, scheme=None, charset=None
    if kwargs["name"] and kwargs["name"] != "Calibri":
        kwargs["scheme"] = None
        kwargs["charset"] = None
        if kwargs["name"] == "quote-cjk-patch":
            kwargs["family"] = None
        else:
            kwargs["family"] = 2  # Helvetica Neue, Arial

    return Font(**kwargs)


def build_fill(fill_spec):
    """Build PatternFill from spec. Returns None если spec пустой."""
    if not fill_spec:
        return None
    pattern = fill_spec.get("pattern", "solid")
    fg, _ = _parse_color(fill_spec.get("fg"))
    bg, _ = _parse_color(fill_spec.get("bg"))
    # openpyxl требует fgColor и bgColor (оба могут быть None → дефолты)
    kwargs = {"patternType": pattern}
    if fg is not None:
        kwargs["fgColor"] = fg
    if bg is not None:
        kwargs["bgColor"] = bg
    return PatternFill(**kwargs)


# Использование:
for sheet_name, sheet_data in spec["sheets"].items():
    ws = wb[sheet_name]
    for addr, cell_spec in sheet_data["cells"].items():
        # Value / formula
        if "formula" in cell_spec:
            ws[addr] = cell_spec["formula"]
        elif cell_spec.get("v") is not None:
            ws[addr] = cell_spec["v"]
        # Style — всегда, даже для пустых ячеек (§12.11)
        ws[addr].font = build_font(cell_spec.get("font"))
        fill = build_fill(cell_spec.get("fill"))
        if fill is not None:
            ws[addr].fill = fill
        # alignment, borders, number_format — аналогично
```

---

#### 12.14.4. Ключевое отличие от «просто Font(...)»

- `build_font(None)` → **полный default** Calibri 12 theme-minor, не пустой шрифт.
- `build_font({"color": "FFF9FAFB"})` → Calibri 12 **без theme-attrs** (RGB разрывает наследование), цвет RGB белый.
- `build_font({"b": true})` → Calibri 12 theme-minor **жирный**.
- `build_fill({"pattern": "solid", "fg": "FF151517"})` → solid-заливка с RGB fgColor.
