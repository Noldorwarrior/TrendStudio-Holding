# ADR-005: Блочный bootstrap годовых YoY — сохранение кризисно-восстановительных пар

**Status:** Accepted
**Date:** 2026-04-10 (v1.3.8)
**Deciders:** ведущий разработчик, риск-менеджер, аналитик рынка проката

## Context

В v1.3.4 была добавлена параметрическая MC симуляция шоков (FX, инфляция, delay) на основе σ, откалиброванных по историческим данным 2014-2023. Это даёт нормальное распределение хвостов, что систематически **недооценивает редкие события**:

- 2020 год (COVID): российский box office рухнул с ~55 млрд ₽ до ~7 млрд ₽ (−87%). Это 6.2σ событие при параметрической калибровке.
- 2022 год (уход Голливуда): 2022 YoY −40% против 2021, затем 2023 YoY +170% (пост-Голливудский всплеск). Пара разнонаправленных шоков со структурной зависимостью.

Параметрический MC не захватывает:
1. Фактические fat tails (парадокс: среднее YoY 2014-2023 = +3%, но σ ≈ 45%).
2. Временную зависимость (кризис-восстановление): после большого падения рынок восстанавливается с высокой вероятностью.
3. Структурные разрывы (COVID, Голливуд) как идентифицируемые события.

### Силы

- **Emergent tail behavior**: распределение EBITDA должно отражать реальные fat tails, а не нормальную аппроксимацию.
- **Структурная зависимость**: bootstrap должен сохранять смежные пары лет (кризис → восстановление), а не перемешивать их независимо.
- **Scarcity of data**: всего 6 YoY-переходов (2019→2020, 2020→2021, ..., 2024→2025) — недостаточно для полноценного распределения, но достаточно для блочного bootstrap.
- **Прозрачность источника**: использовать ЕАИС (Единая автоматизированная информационная система) — официальный российский источник.

## Decision

Внедрить **block bootstrap** годовых YoY множителей рынка проката, с block_size=2, в `generators/market_bootstrap.py`.

### Алгоритм

1. Загрузить `inputs/eais_seed/annual_box_office.csv` (2019-2025, 7 точек, 6 YoY-переходов).
2. Для каждой из n_simulations=2000 симуляций:
   - Выбрать случайный стартовый индекс i ∈ [0, 6 − block_size].
   - Взять блок YoY[i : i+block_size].
   - Если horizon_years=3 > block_size=2, дополнить блок независимыми выборками из оставшихся переходов.
   - Применить yoy_t к `cinema.targets_mln_rub[t]` через `market_beta`:
     ```
     scale_t = 1.0 + market_beta · (yoy_t − 1.0)
     target_t_rescaled = target_t · scale_t
     ```
3. Прогнать `run_all` с изменёнными targets, собрать `cumulative_ebitda`.
4. Рассчитать mean, p5/p25/p50/p75/p95, VaR95, breach_p.

### Параметры

- `block_size=2` — сохраняет пары типа (COVID trough 2020, post-COVID rebound 2021) и (Hollywood exit 2022, russification 2023).
- `market_beta=0.40` — доля cinema.targets, чувствительная к рыночному YoY. Консервативная оценка: ~60% доходов — долгосрочные контракты и SVOD, нечувствительные к YoY рынка.
- `seed=43` — независим от parametric MC (seed=42).
- `exclude_years=[]` — используем все 6 YoY переходов, включая COVID и Голливуд (это целевые fat tails).

## Options Considered

### Option A: iid bootstrap (без блоков)

| Dimension | Assessment |
|-----------|------------|
| Complexity | Low |
| Cost | Low |
| Scalability | High |
| Team familiarity | High |

**Pros:** простота, теоретическая корректность при iid предположении.
**Cons:** **разрушает** временную зависимость. Симуляция типа (COVID −87%, COVID −87%, Голливуд −40%) физически невозможна, но iid bootstrap её допускает. Получаем слишком тяжёлые хвосты.

### Option B: Block bootstrap block_size=2 (выбрано)

| Dimension | Assessment |
|-----------|------------|
| Complexity | Low |
| Cost | Low |
| Scalability | High |
| Team familiarity | Med |

**Pros:** сохраняет пары (спад, восстановление). Реалистичные trajectory — большинство блоков физически возможны. 2-летний блок покрывает среднюю длительность кризиса на российском рынке проката (COVID restrictions ≈ 14 мес, Hollywood exit ≈ 12-18 мес).
**Cons:** остаётся небольшая non-iid зависимость между блоками при дополнении horizon=3.

### Option C: Block bootstrap block_size=3 (полный horizon)

| Dimension | Assessment |
|-----------|------------|
| Complexity | Low |
| Cost | Low |
| Scalability | Low (только 4 возможных стартовых индекса) |
| Team familiarity | Med |

**Pros:** максимальная сохранность временной структуры.
**Cons:** при block_size=3 и 6 переходах имеется всего **4** возможных блока (i ∈ {0,1,2,3}). Это критически мало — распределение EBITDA получается дискретным с 4 модами. Variance estimator нестабильна.

### Option D: Parametric t-distribution (фат хвосты)

| Dimension | Assessment |
|-----------|------------|
| Complexity | Med |
| Cost | Med (нужна калибровка df) |
| Scalability | High |
| Team familiarity | Low |

**Pros:** аналитическая формула, гладкое распределение.
**Cons:** теряет идентифицируемость структурных событий (COVID, Hollywood). Калибровка df на 6 точках ненадёжна.

## Trade-off Analysis

Option B выбран как минимальный блочный bootstrap, сохраняющий временную структуру без коллапса комбинаторики. Расчёт: block_size=2 даёт `6 − 2 + 1 = 5` возможных стартовых позиций, каждая — со своим физическим смыслом:

| Start | Блок | Смысл |
|---|---|---|
| 0 | 2019→2020, 2020→2021 | COVID trough + rebound |
| 1 | 2020→2021, 2021→2022 | Post-COVID normalization + gradual decline |
| 2 | 2021→2022, 2022→2023 | Hollywood exit + transition |
| 3 | 2022→2023, 2023→2024 | Russification phase 1 + 2 |
| 4 | 2023→2024, 2024→2025 | Steady russification |

5 стартов × 2000 sims даёт равномерное покрытие всех реалистичных сценариев.

Option A отвергнут из-за разрушения временной структуры. Option C отвергнут из-за дискретности с 4 модами. Option D отвергнут из-за потери идентификации событий.

## Consequences

**Становится проще:**
- Tail-aware распределение: p5 и breach_probability отражают реальные экстремальные сценарии, а не нормальную аппроксимацию.
- Явное обоснование для риск-менеджера: «3% симуляций содержат пару типа COVID, что соответствует вероятности нового шока ≈ 1 раз в 30 лет».
- Возможность включать/исключать структурные разрывы через `exclude_years: [2020, 2022]` для sensitivity.

**Становится сложнее:**
- Результаты зависят от `market_beta` — чувствительность к этому параметру велика, потребуется sensitivity analysis.
- При обновлении eais_seed CSV (новый год) — границы блоков меняются, нужна регрессия.
- Нельзя напрямую сравнивать с parametric MC — распределения имеют разную природу.

**Надо пересматривать:**
- При добавлении наблюдений 2026+ — пересчитать все результаты с block_size=2 (≥ 7 возможных блоков).
- Если появится более глубокая история (2015+) — рассмотреть block_size=3.
- При изменении структуры рынка (возвращение Голливуда, новый шок) — пересмотреть `exclude_years`.
