### 6.1. Обнаруженная openpyxl-ловушка №1 — default font

```
До фикса:
  DEBUG: Before save, PnL A2 size = 12.0
  DEBUG: After save/load, PnL A2 size = 11.0
  Rebuilt has 3 distinct fonts:
    [0] Calibri 11 + theme color   ← куда попадает A2
    [1] Calibri 12
    [2] Calibri 12 + theme color
```

При создании пустого `Workbook()` openpyxl инициализирует `wb._fonts[0] = Font(name="Calibri", size=11)`. Если на ячейку назначить `Font(name="Calibri", size=12, color=Color(theme=1))`, то при сохранении openpyxl дедуплицирует стили и **назначает ячейке индекс font[0] (с размером 11)**, теряя явно указанный размер 12. Это происходит только для пустых ячеек с theme-color шрифтом — ячейки со значениями сохраняют стиль корректно.

**Workaround:** заменить `wb._fonts[0]` и `wb._named_styles['Normal'].font` на `Font(name="Calibri", size=12, family=2, scheme="minor", charset=204, color=Color(theme=1))` **до** создания любых ячеек.

### 6.2. Обнаруженная openpyxl-ловушка №2 — theme-font inheritance

В оригинале Calibri-ячейки с theme-color имеют неочевидные атрибуты: `font.scheme='minor'`, `font.family=2`, `font.charset=204`. Эти атрибуты Excel генерирует автоматически, когда применяется theme-color. Если исполнитель создаёт `Font(name="Calibri", size=12, color=Color(theme=1))` без этих атрибутов — файл визуально отличается в Excel (theme font lookup использует другой шрифт).

**Правила inheritance** (обнаружены стресс-тестом Слоя 5):

| Шрифт | Цвет | family | scheme | charset |
|-------|------|--------|--------|---------|
| Calibri | theme | 2 | minor | 204 |
| Calibri | нет цвета | 2 | minor | 204 |
| Calibri | RGB | None | None | None |
| Arial | любой | 2 | None | None |
| Helvetica Neue | любой | 2 | None | None |
| quote-cjk-patch | любой | None | None | None |

Правило: **theme-цвет наследует theme-font-атрибуты**, **RGB-цвет разрывает наследование**.
